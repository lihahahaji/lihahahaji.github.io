# Dijkstra

# 算法特点

  • 求解单源最短路问题
  • 可以求解从源点到图上其他所有点的距离，不需要有特定的目标点
  • 要求图中不存在负权边

# 算法实现

首先我们做两个定义：

  • 维护一个 dis 数组，dis[i] 表示节点 i 到源点的最短距离，初始状态下，dis[0]=0，其他节点的 dis 值都为 ∞

  • 定义两个集合:

    • 开放列表（OpenList）：已经访问过，但是没有确定最短路径的节点。
    • 关闭列表（CloseList）：已经访问过，并且已经确定了源点到其的最短路径的节点。

# 算法流程

  1. 将起点 S 放入开放列表 (openList) 中.

  2. 遍历开放列表

    1. 计算列表中的每一个节点 n 的dis(n) , 查找dis(n) 值最小的节点, 将这个节点作为当前要处理的节点N

    2. 对于当前正在处理的节点 N , 遍历与之相邻的其他所有节点 Nnear​

      • 假设当前正在处理的相邻节点为 Nnear​

        1. node.state = close Nnear​ 已经在关闭列表中（已经确定最短路径），或者不可达（障碍物），则忽略该节点

        2. node.state = new Nnear​ 不在开放列表中（未被访问过），则将其加入开放列表，将当前节点 N 设置为 Nnear​ 的父结点，计算节点 Nnear​ 的 dis(Nnear​)=dis(N)+distance(N,Nnear​)

        3. node.state = open Nnear​ 已经在开放列表中（已经访问过但未确定最短路径），则需要检查这条路径（节点 N 到节点 Nnear​ 的路径）是否更好，参考指标为 g(Nnear​) ,若g(Nnear​) 更小，则说明该路径更好，则将它的父亲节点（设为节点）设置为当前节点 N ，并重新计算f(Nnear​) 。

          dis(Nnear​)=min{ dis(N)+distace(N,Nnear​) , dis(Nnear​) }

    3. 当前节点 N 处理完毕， 移动到关闭列表，表示该节点已经不需要被继续关注。

  3. 重复步骤「2」直到满足以下条件之一，算法结束：

    1. 开放列表为空（已经没有可以访问的点）

  4. 路径回溯：从终点开始，每一个节点都沿着父亲节点移动，直到找到起点，即可得到一条路径。

# 代码实现

根据算法的特性，我们可以知道，OpenList 是一个集合，CloseList 是一个栈（用集合也可以）。

C++ 代码实现如下：

# 节点数据结构

// 定义节点的结构
struct Node
{
    // 当前节点的状态
    // 0 - new
    // 1 - open
    // -1 - close
    int state;

    // 当前节点的坐标值
    int x;
    int y;

    // 当前节点的 h 值（也就是 dijkstra 中的 dis ）
    double h;

    // 当前节点的 k 值
    double k;

    // 当前节点指向的父节点
    int father;

    // 当前节点的下标
    int index;
};

# 核心代码

// 使用 dijkstra 规划最短路径
void dijkstra(int s, int e)
{
    // 将起点 S 放入开放列表
    OpenList.insert(&node[s]);
    node[s].state = 1;

    while (OpenList.size())
    {
        // 遍历 OpenList，找到 h 值最小的节点 n
        int min_h_value = INF;
        // 找到当前正在处理的节点的指针
        Node *N_ptr;
        for (set<Node *>::iterator i = OpenList.begin(); i != OpenList.end(); i++)
        {
            Node *nodePtr = *i;
            if (nodePtr->h < min_h_value)
            {
                min_h_value = nodePtr->h;
                N_ptr = nodePtr;
            }
        }

        // 遍历 N 的邻接节点
        for (int i = 0; i < 8; i++)
        {
            // 先判断当前节点是否存在（合法）
            int n_near_x = N_ptr->x + directions[i][0];
            int n_near_y = N_ptr->y + directions[i][1];
            if (!isValidNode(n_near_x, n_near_y))
                continue;

            Node *N_near_Ptr = &node[n_near_x * n + n_near_y];

            // 当前节点的状态为 new（没有访问过）
            if (N_near_Ptr->state == 0)
            {
                // 加入开放列表
                OpenList.insert(N_near_Ptr);
                N_near_Ptr->state = 1;
                // 更新 h 值
                N_near_Ptr->h = N_ptr->h + (i > 3 ? 1.4 : 1);
                // 将 n 设置为 n_near 的父结点
                N_near_Ptr->father = N_ptr->index;
            }
            // 当前节点的状态为 open（已经访问过但是还没有确定最短路径）
            else if (N_near_Ptr->state == 1)
            {
                double new_h = N_ptr->h + (i > 3 ? 1.4 : 1);
                if (N_near_Ptr->h > new_h)
                {
                    N_near_Ptr->h = new_h;
                    N_near_Ptr->father = N_ptr->index;
                }
            }
            // 当前节点处理完毕，移入CloseList
            N_ptr->state = -1;
            OpenList.erase(N_ptr);
            CloseList.insert(N_ptr);
            // 终点进入 CloseList 算法结束
            if (N_ptr->index == e)
                return;
        }
    }
}
